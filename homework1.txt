1. Как скопировать на свой компьютер репозиторий из github.com, что должно быть сделанно предварительно?

Создать SSH ключи, публичный загрузить в github. 
Создать в github реппозиторий (public, Add a README file), зайти в него, в списке Code выбрать SSH, скопировать.
На локальном компьютере в git-bush зайти (создать) папку, в которой собираемся работать, ввсести git clone + вставить скопированную ссылку

2. Как создать git репозиторий из локального каталога на своем комьютере. Как его отправить на github.com?

Создать реппозиторий локально, в git-bush зайти в соотв.дирректорию и ввести git init
Проверить, какие файлы не добавлены под контроль git: git status
Проиндексировать нужные файлы: git add . 
Закоммитить файлы: git commit -m "Added new files"
На github создать новый реппозиторий (без README file)
Создать ветку main: git branch -M main
Добавить удаленное хранилище: git remote add origin git@github.com:SvitlanaMykytiuk/Test.git
Проверить, подключилось ли удаленное хранилище: git remote -v
git push -u origin main

3. Как получить на локальный компьютер изменения с github.com

На github в режиме редактора внести изменения в файл
Подтвердить изменения: Commit changes
На локальном компьютере получить изменения: git pull

Какую информацию нам дают команда :
git status  - показывает состояние файлов по отношению к git (закоммичены и т.д) 
git log - показывает коммиты от самого нового к самому первому, кто внес изменения, комментарий
git remote -v  - выдает ссылку на реппозиторий в github
